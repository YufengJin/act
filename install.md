## Host prerequisites

- Docker (>= 24) and the Compose v2 plugin
- NVIDIA driver >= 450 and the NVIDIA Container Toolkit
- A GPU visible to `nvidia-smi`

Run the bundled check before building:

```bash
bash docker/check_prereqs.sh
```

## Build the image

```bash
docker compose -f docker/docker-compose.headless.yaml build
```

Base image: `nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu22.04`
Python: `3.9` (managed by `uv` in `/opt/venv`)
Install strategy: `uv sync --frozen --no-install-project --no-install-workspace --active`

Key notes:
- `--active` flag ensures packages install into `/opt/venv` (not a project-local `.venv`)
- `detr` is a uv workspace member; its `pyproject.toml` is COPY-ed at build time
- Editable install of both `act` and `detr` runs in `entrypoint.sh` on container start

## Bring the container up

```bash
docker compose -f docker/docker-compose.headless.yaml up -d
docker exec -it act-headless bash
```

For an X11-enabled session (GUI apps, visualisers):

```bash
xhost +local:docker
docker compose -f docker/docker-compose.x11.yaml up -d
```

Only one of the two profiles may run at a time — they share the container name.

## First-run sanity checks (inside the container)

```bash
nvidia-smi
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
python -c "import sim_env; print(sim_env.__file__)"
```

The last line must resolve under `/workspace/act/` — this proves that the mounted source is the one the interpreter imports (editable install succeeded). `act` does not expose a top-level package; the canonical proof-of-import is `import sim_env` (the bimanual MuJoCo env factory module).
